Chaining modifiers

The output of one modifer can be used as the input for the next one. Seen the movie The Human Centipede? It's a bit like that. In this latest contrived example, we're going to use chaining to make a colour mixer.

First, we want to show how much we're using of each colour. We'll use <div> elements with a percentage width corresponding to the amount. All the colour values are between 0 and 1, so we can reuse our percentage-ifier from the last step:

<td><div style='background-color: red;
    	        width: {{red}}%;'></div></td>

Update the first three <div> elements in the template so they use the multiply modifier.

To show the result, we can use an rgb(r,g,b) CSS colour value. But instead of percentages, these need to be between 0-255:

<div style='background-color: rgb({{red   | multiply[255] }},
    	                          {{green | multiply[255] }},
    	                          {{blue  | multiply[255] }})'></div>

Update the template and execute the code. Did it work?

No, it didn't. That's because CSS insists that you use integers – no decimals allowed. So let's create a round modifier to turn the numbers into integers:

function ( num ) {
  return Math.round( num );
}

Add that, then add the modifier to the template, then execute the code.

Try changing the colours using the console. For extra credit, try adding a clamp modifier which ensures the input values are clamped to between 0 and 1 before they get multiplied.
