Modifiers with arguments

You can specify arguments for modifiers by placing them in square brackets. In the example below, we've correctly multiplied 0.5 by 100 to get an answer of 50%, but 100 is a magic number, so we should try to remove it.

First, let's add 100 as an argument for our multiply modifier:

{{num | multiply[100] }}

Then, we change the modifier so it receives the argument:

function ( num, multiplier ) {
  return num * multiplier;
}

Go ahead and edit the code. When you execute it, nothing should change, except that fewer kittens will die as a result of you using magic numbers.

You can have as many arguments as you like, but they must be parsable by JSON.parse. That means that formatter[1,2,3] or formatter["some text"] are valid argument sets, but formatter['some text'] is not (strings must be double quoted), and neither is formatter[someVariable] since variable names are not valid JSON.
