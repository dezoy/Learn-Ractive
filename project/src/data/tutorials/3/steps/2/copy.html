Unsubscribing from events

There are a couple of ways to unsubscribe from events. If you've used jQuery, you'll be used to this syntax:

view.on( 'select', selectHandler );

// later...
view.off( 'select', selectHandler );

That's fine, as long as you stored a reference to selectHandler (i.e. you didn't just use an anonymous function). If you didn't, you can also do this:

view.off( 'select' ); // unsubscribes ALL 'select' handlers
view.off(); // unsubscribes all handlers of any type

Alternatively, you can do this:

var listener = view.on( 'select', selectHandler );

var otherListeners = view.on({
  activate: function () { alert( 'Activating' ); },
  dectivate: function () { alert( 'Deactivating!' ); }
});

// later...
listener.cancel();
otherListeners.cancel();

Try adding a 'silence' button which removes the 'activate' and 'deactivate' handlers.

We haven't yet covered view.teardown(), which removes your view from the DOM and cleans up after it. However as you'd expect this will remove any event handlers bound with view.on().
